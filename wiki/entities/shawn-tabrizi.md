## Core thesis: "Polkadot Cloud"

Tabrizi coined and evangelized the **"Polkadot Cloud"** framing in late 2024, positioning Polkadot not as a multi-chain blockchain network but as a decentralized cloud platform:

> *"JAM is the beginning of the Polkadot cloud."*

Key arguments:
- Current blockchains execute code only within single block-time windows — fundamentally limiting application design
- JAM breaks this constraint by enabling asynchronous, long-running computation
- The Polkadot Cloud offers decentralized execution, settlement, data availability, and object storage — functions analogous to AWS/Azure but cryptographically trustless
- JAM's PVM (Polkadot Virtual Machine), based on RISC-V, abstracts blockchain complexity so developers can "build an application, deploy it to our cloud and we'll split it up for you" automatically
- Polkadot Cloud specs: 1,000+ validators, 341 cores, 1M+ TPS

Core vision statement:
> *"JAM evolves Polkadot from hosting just blockchains to a platform for hosting any web3 application or service."*
> *"Polkadot is not just a blockchain. It's a vision toward a world with less trust and more truth."*

This "Polkadot Cloud" framing was presented as a community discussion (not locked-in official positioning) but became influential in shaping the "Second Era" narrative.
